# 产线控制后端 V0

## 项目概览
本仓库用于实现《[PRD]产线控制软件V0》后端，技术栈为 `FastAPI + Pydantic + SQLite + SQLModel`。

当前代码库已包含以下模块：
- 生产任务导入：`orders`
- lot 管理：`lots`
- AI 辅助 lot 规划：`ai`
- 网版设计：`process_plan`
- 工艺路线设计：`process_route`

当前代码库已包含多个并行模块，其中网版设计 `process_plan` 保持可用，工艺路线设计 `process_route` 已新增接口。`README.md` 负责描述项目当前已有 API；`AGENTS.md` 负责描述当前阶段优先收敛的实现目标、约束和验收重点。

## 快速开始
本地启动：

```powershell
cd backend
python -m venv .venv_windows
.\.venv_windows\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python -m uvicorn app.main:app --host 127.0.0.1 --port 18000
```

常用地址：
- 健康检查：`http://127.0.0.1:18000/healthz`
- OpenAPI：`http://127.0.0.1:18000/docs`

在 PowerShell 中调用 `curl` 时，建议使用 `curl.exe`，避免被解析为 `Invoke-WebRequest`。

## API 总览

### 系统接口
- `GET /healthz`
  - 轻量健康检查
  - 返回 `{"ok": true}`

### 生产任务导入 `orders`
- `GET /orders/list`
  - 查询已导入生产任务列表
- `GET /orders/{order_id}`
  - 查询单个生产任务详情
- `POST /orders/import_local`
  - 导入本地生产任务 XML
- `POST /orders/{order_id}/validate`
  - 对已导入生产任务做显式校验，并在通过时更新状态

### Lot 管理 `lots`
- `GET /lots/list`
  - 查询 lot 列表
- `GET /lots/{lot_id}`
  - 查询单个 lot 详情
- `POST /lots/import_lines`
  - 从生产任务行导入到 lot
- `POST /lots/{lot_id}/validate`
  - 对 lot 做显式校验，并在通过时更新状态

### AI 辅助 `ai`
- `POST /ai/optimize_lots`
  - 根据选定订单行生成 lot 建议，`dry_run=false` 时可直接落库
- `POST /ai/validate_pending_lots`
  - 批量校验当前未 `validated` 的 lot

### 网版设计 `process_plan`
- `GET /process_plan/list`
  - 查询历史工艺方案列表
- `GET /process_plan/{process_plan_id}-{process_plan_version}`
  - 查询指定版本工艺方案详情
- `POST /process_plan/validate`
  - 对上传的工艺方案做显式校验，不落库
- `POST /process_plan/approve`
  - 校验通过后批准并落库工艺方案

### 工艺路线设计
- `GET /process_route/list`
  - 获取所有历史工艺路线版本
  - 直接返回数组，元素包含 `lot_id`、`process_route_id`、`process_route_version`、`line_spec_id`、`approved_by`、`status`
- `GET /process_route/{process_route_id}/{process_route_version}`
  - 获取特定工艺路线详情
  - 返回 `{process_router_header, process_route_loop_line, process_route_loop_step}`
- `POST /process_route/validate`
  - 把正在编辑的工艺路线方案提交给后端校验，不落库
  - 请求体使用 `process_route_header`、`process_route_loop_line`、`process_route_loop_step_line`
  - 返回 `passed`、`errors`、`risks`，其中 `errors` 和 `risks` 为字符串列表
- `POST /process_route/approve`
  - 校验通过后冻结并落库当前工艺路线方案
  - 返回 `passed`、`errors`、`risks`、`process_route_id`、`process_route_version`

工艺路线状态说明：
- 设计文档中的 `frozen` 在后端持久化状态中统一映射为 `validated`
- 后端当前固定状态集合为 `created`、`validated`、`obsolete`
