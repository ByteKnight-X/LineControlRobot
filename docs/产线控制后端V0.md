# 产线控制后端 V0

本仓库用于实现《[PRD]产线控制软件V0》后端。当前阶段以《[DesignDoc]生产任务导入B0》定义的 B0 生产任务导入闭环能力为准。

## 当前范围（B0）
- 导入生产订单 XML
- 校验生产订单完整性
- 从订单行创建/补充 lot
- 校验未完成 lot
- 提供 AI lot 拆分/校验的 mock 能力
- 保持接口可本地跑通并可测试

## 快速开始
```powershell
cd backend
python -m venv .venv_windows
.\.venv_windows\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python -m uvicorn app.main:app --host 127.0.0.1 --port 18000
```

健康检查：`curl http://127.0.0.1:18000/healthz`  
OpenAPI：`http://127.0.0.1:18000/docs`

## B0 接口清单
- `GET /orders/list`
- `GET /orders/{order_id}`
- `POST /orders/import_local`
- `POST /orders/{order_id}/validate`
- `GET /lots/list`
- `GET /lots/{lot_id}`
- `POST /lots/import_lines`
- `POST /lots/{lot_id}/validate`
- `POST /ai/optimize_lots`
- `POST /ai/validate_pending_lots`

## B0 接口约定
### `POST /orders/import_local`
- 请求体：`application/xml`
- 输入：生产订单 XML，根节点为 `production_order`
- 成功响应：

```json
{
  "order_id": "PO-20260206-01",
  "status": "created",
  "line_count": 2
}
```

### `POST /lots/import_lines`
- 请求体：JSON
- `lot_id` 可选；缺省时自动新建批次

```json
{
  "order_id": "PO-20260206-01",
  "selected_order_line_ids": [1, 2]
}
```

### `POST /lots/{lot_id}/validate`
- 校验通过后 lot `status` 更新为 `validated`
- 校验失败时 lot `status` 保持 `created`
- 响应包含 `passed`、`errors`、`risks`

### `POST /ai/optimize_lots`
- `dry_run=true`：只返回 `proposal`，不落库
- `dry_run=false`：直接创建 lot 并落库

### `POST /ai/validate_pending_lots`
- 仅扫描 `status != validated` 的批次
- 返回 `results: [{lot_id, passed, errors, risks}]`
- `passed=true` 的批次会更新为 `validated`
